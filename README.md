# Northwind Traders — Analytics Challenge

## Objective

Technical challenge for the **Analytics Engineer** position at Indicium. The goal is to demonstrate the ability to build a complete data pipeline — from ingesting data into Bronze/Silver/Gold layers on Databricks, to delivering an executive report in Power BI and Python — using historical data from the fictional company Northwind Traders (a food distributor).

## Data Pipeline (Databricks)

The architecture follows the **Medallion** pattern across three layers:

| Layer | Notebook | What it does |
|-------|----------|--------------|
| **Bronze** | `01_bronze_ingestion.ipynb` | Reads CSVs from the DBFS volume and saves them as Delta with no transformations, adding `ingestion_ts` and `source_file` metadata columns |
| **Silver** | `02_silver_transformations.ipynb` | Applies typing, quality filters and computes derived metrics (`gross_sales`, `net_sales`, `shipped_on_time_flag`, `shipping_delay_days`) |
| **Gold** | `03_gold_star_schema.ipynb` | Builds the star schema: denormalised dimensions (`dim_product` with inline category, `dim_territory` with inline region, `fact_sales` with inline shipper and employee) and a programmatically generated `dim_calendar` |

## Power BI Dashboard

The report has **5 pages**:

1. **Executive Overview** — Main KPIs, monthly revenue and revenue by category
2. **Avg Order Value & Product Mix** — Average order value, scatter (Orders × AOV), top 15 products
3. **Customers and Churn** — Retention, new customers, at-risk customers, inactivity distribution
4. **Commercial Performance** — Salesperson ranking, performance by territory
5. **Logistics Efficiency & Service Level** — % on-time deliveries, delay distribution, service-level gauge

The semantic model uses the **Databricks Gold schema** as source via native connector, with DAX measures computed in the `calculations` layer.

## Executive Report (PDF)

The notebook `analise_northwind_databricks.ipynb` connects to Databricks via SQL, reproduces the analyses from all 5 Power BI pages in Python (Plotly), and generates `analise_northwind_executivo.pdf` — an executive document with KPIs and 8 static charts, ready for presentation.

To regenerate the PDF, simply run all notebook cells in order. Databricks connection parameters are loaded from the `.env` file (see `.env` for the required variables).
